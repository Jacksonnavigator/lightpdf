1
Push Your Code to GitHub
Render deploys directly from GitHub — this is how it works
ℹ️
If you don't have a GitHub account, create one free at github.com. It only takes 2 minutes.

A) Create a new GitHub repository

1
Go to github.com → click the "+" button top-right → "New repository"
2
Name it something like pdf-editor
3
Set it to Private (recommended) or Public
4
Do NOT add README or .gitignore — keep it empty
5
Click "Create repository"

B) Upload your code from your computer

Open a terminal on your computer, navigate to the PdfRender folder from the zip you downloaded, and run:

Terminal — Mac/Linux
# Navigate into the PdfRender folder
cd ~/Downloads/PdfRender

# Initialize git and push to GitHub
git init
git add .
git commit -m "Initial commit — PDF editor with Collabora"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/pdf-editor.git
git push -u origin main
Terminal — Windows PowerShell
# Navigate into the PdfRender folder
cd C:\Users\YourName\Downloads\PdfRender

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/pdf-editor.git
git push -u origin main
💡
When asked for GitHub credentials, use your GitHub username and a Personal Access Token (not your password). Create one at: GitHub → Settings → Developer settings → Personal access tokens → Generate new token → select "repo" scope.
✅ Success looks like: After pushing, refresh your GitHub repository page — you should see all your files: backend/, frontend/, collabora/, render.yaml, etc.
2
Create Your Render Account & Connect GitHub
render.com — free, no credit card needed for free tier
1
Go to render.com → click "Get Started for Free"
2
Sign up using your GitHub account (easiest — click "GitHub" button)
3
Authorize Render to access your GitHub
4
You'll land on the Render Dashboard — a clean page with "New +" button at the top
💡
Signing up with GitHub automatically links your repos — Render can see and deploy from them immediately.
3
Deploy the Backend Service
Node.js + LibreOffice — the brain of your app
1
On Render Dashboard → click "New +" → "Web Service"
2
Click "Connect a repository" → select your pdf-editor GitHub repo → click "Connect"

Fill in these settings:

FieldValue
Namepdf-backend
RegionChoose closest to you
Branchmain
RuntimeDocker
Dockerfile Path./backend/Dockerfile
Docker Context./backend
PlanFree

⚠️ Add Environment Variables (scroll down on same page)

Click "Advanced" → then "Add Environment Variable" for each one below:

PORT
3000
CONVERT_MODE
local  ← very important, don't skip this
DATA_DIR
/data
CONVERT_TIMEOUT_MS
180000
BASE_URL
Leave blank for now — fill in after deploy (see Step 5)
COLLABORA_URL
Leave blank for now — fill in after deploy (see Step 5)

First build takes 8–15 minutes because it installs LibreOffice inside the container. This is normal! You'll see a build log streaming — wait for it to say "Your service is live".
✅ When done, note your backend URL — it looks like: https://pdf-backend-xxxx.onrender.com
Copy this URL — you'll need it in Steps 4 and 5.

5
Deploy Frontend & Update URLs
Connect everything together

A) Update Backend with the real URLs

Now that you have both URLs, go back to your pdf-backend service on Render:

1
Click on pdf-backend → click "Environment" tab on the left
2
Update these two variables:
BASE_URL
https://pdf-backend-xxxx.onrender.com (your backend's own URL)
COLLABORA_URL
https://pdf-collabora-xxxx.onrender.com (your Collabora URL)
3
Click "Save Changes" — backend will redeploy automatically (1–2 min)

B) Deploy the Frontend

1
Dashboard → "New +" → "Web Service"
2
Select your pdf-editor repo → "Connect"
FieldValue
Namepdf-frontend
RuntimeDocker
Dockerfile Path./frontend/Dockerfile
Docker Context./frontend
PlanFree

Add Build Argument (not env var — it's a build arg):

Click "Advanced" → look for "Build Arguments" section:

VITE_API_URL
https://pdf-backend-xxxx.onrender.com
3
Click "Create Web Service" — wait for build to finish
✅ When done, your frontend URL looks like: https://pdf-frontend-xxxx.onrender.com
This is the link you share with everyone!
6
Test Your Live App 🎉
Make sure everything works end to end

Quick health check first

Open this URL in your browser (replace with your real backend URL):

https://pdf-backend-xxxx.onrender.com/api/health

You should see: {"ok":true} — if yes, backend is working ✅

Full test

  • Open your frontend URL: https://pdf-frontend-xxxx.onrender.com
  • Upload a PDF file
  • Wait ~15–30 seconds for conversion (first time may be slower as service wakes up)
  • Click Open Editor — Collabora loads in the browser
  • Edit some text, then press Ctrl+S to save
  • Click Download PDF or Download DOCX

Troubleshooting

❌
App is slow / not responding — Free tier sleeps after 15 min. First request after sleep takes 20–30 seconds to wake up. Just wait and refresh.
❌
Editor doesn't open / blank screen — Check that COLLABORA_URL env var in the backend is set correctly. Go to pdf-backend → Environment → verify the URL matches your pdf-collabora service URL exactly.
❌
Conversion fails — Check that CONVERT_MODE=local is set. Go to pdf-backend → Environment and verify it.
❌
"Download PDF" says save first — You must press Ctrl+S inside the Collabora editor to trigger the save, then download.
💡
To see live logs for any service: click the service on Render Dashboard → click "Logs" tab on the left sidebar. This shows exactly what's happening.

2
Connect to Server & Install Docker
SSH in and get Docker running
SSH IN (Mac/Linux)
chmod 400 ~/Downloads/ssh-key.key
ssh -i ~/Downloads/ssh-key.key ubuntu@YOUR_IP
SSH IN (Windows PowerShell)
ssh -i "C:\Users\You\Downloads\ssh-key.key" ubuntu@YOUR_IP
INSTALL DOCKER (run inside SSH)
# Update system
sudo apt-get update && sudo apt-get upgrade -y

# Install Docker
curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh

# Add user to docker group
sudo usermod -aG docker ubuntu && newgrp docker

# Install Docker Compose
sudo apt-get install -y docker-compose-plugin

# Verify
docker --version && docker compose version
OPEN FIREWALL PORTS
sudo ufw allow 22 && sudo ufw allow 80
sudo ufw allow 3000 && sudo ufw allow 9980
sudo ufw --force enable
⚠️
Also open ports 80, 3000, 9980 in Oracle Cloud's Security List (same steps as before — Networking → VCN → Security Lists → Add Ingress Rules).
3
Upload Your Updated Code
From your local machine to the server

The zip file you downloaded contains your complete updated project. Upload it:

FROM YOUR LOCAL COMPUTER
# Mac/Linux:
scp -i ~/Downloads/ssh-key.key -r ~/Downloads/PdfApp ubuntu@YOUR_IP:~/

# Windows PowerShell:
scp -i "C:\Users\You\Downloads\ssh-key.key" -r "C:\Users\You\Downloads\PdfApp" ubuntu@YOUR_IP:~/
💡
Alternatively, push to GitHub and run git clone https://github.com/yourname/yourrepo.git PdfApp on the server — easier for future updates.
4
Configure Your IP Address
The most important step — do this carefully

In your SSH terminal, navigate to your project and set your real IP in 2 files:

NAVIGATE TO PROJECT
cd ~/PdfApp

Update docker-compose.yml

Run this command — replace YOUR_ACTUAL_IP with your real Oracle/server IP:

sed -i 's/YOUR_IP/YOUR_ACTUAL_IP/g' docker-compose.yml

# Verify it was replaced correctly:
grep "YOUR_IP" docker-compose.yml  # Should return nothing if replaced
grep "BASE_URL" docker-compose.yml  # Should show your real IP

Update backend .env

cat > backend/.env << 'EOF'
PORT=3000
BASE_URL=http://YOUR_ACTUAL_IP:3000
COLLABORA_URL=http://YOUR_ACTUAL_IP:9980
DATA_DIR=/data
CONVERT_MODE=docker
LIBREOFFICE_CONTAINER=pdf-libreoffice
EOF

Create the data directory

mkdir -p data && chmod 755 data
⚠️
The IP must be your public IP (the one shown on Oracle Cloud Console), not localhost or a private IP. Collabora needs to reach your backend from inside Docker using this IP.
5
Launch the App!
Build and start all containers
BUILD & START (first time — takes 5–10 min)
docker compose up -d --build
WATCH STARTUP LOGS
docker compose logs -f

# Press Ctrl+C to stop watching — app keeps running
CHECK ALL SERVICES ARE UP
docker compose ps

# You should see 4 services all showing "Up" or "running":
pdf-backend      Up
pdf-collabora    Up
pdf-frontend     Up
pdf-libreoffice  Up
⏳
Collabora takes about 60–90 seconds to fully start. Wait for it before testing. You can watch with: docker compose logs -f collabora
6
Test & Use Your App 🎉
Open in browser and try it out

Open your browser and visit:

http://YOUR_IP           # ← Your app (main page)
http://YOUR_IP:3000/api/health  # ← Should show {"ok":true}

How to use the app

  • Upload a PDF — it converts to DOCX in the background (~10–30 seconds)
  • Click Open Editor — Collabora loads the Word-like editor in the browser
  • Edit text, images, formatting — feels just like Word
  • Click File → Save (or Ctrl+S) inside the editor to save changes
  • Click Download PDF or Download DOCX in the toolbar

Useful commands

# Stop everything
docker compose down

# Restart everything
docker compose restart

# Update after code changes
git pull && docker compose up -d --build

# View logs for a specific service
docker compose logs backend
docker compose logs collabora

# Free up disk space
docker system prune -f

Troubleshooting

Editor doesn't load? — Check Collabora started: docker compose logs collabora. Wait 90 seconds after startup.

Can't connect to app? — Verify ports 80, 3000, 9980 are open in Oracle Security List AND ufw.

Conversion fails? — Check docker compose logs backend and make sure data/ folder exists with write permission.

Download says "save first"? — You must click Save (Ctrl+S) inside the Collabora editor before downloading.
